# tuile

A reusable TUI component framework built on ratatui. Inspired by tui-realm,
but with concrete types instead of runtime-typed enums, a one-hop event model,
and cascading themes with built-in contrast validation.

The name is French for "tile" (also a thin curved French cookie) — it contains
"tui" naturally, mirroring how "ratatui" comes from "ratatouille" + "tui."

Status: early development. API is unstable.
